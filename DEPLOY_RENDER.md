# نشر بابل AI على Render

هذه هي طريقة النشر كموقع عام أولي. لا ترفع `.env.local` ولا تضع مفتاح OpenAI داخل الملفات.

## 1. قبل النشر

تأكد من هذه النقاط:

- رصيد OpenAI API يعمل.
- تم إنشاء مفتاح OpenAI جديد ونظيف.
- كود الوصول ليس بسيطا مثل `arkan2026` عند النشر العام.
- `.env.local` موجود محليا فقط وغير مرفوع.

## 2. ملفات النشر الجاهزة

تمت إضافة:

- `server.js`: خادم Node.js قابل للاستضافة.
- `package.json`: أمر تشغيل الموقع.
- `render.yaml`: إعداد أولي لـ Render.
- `.env.example`: مثال للمتغيرات بدون أسرار.

## 3. متغيرات البيئة المطلوبة في Render

في إعدادات الخدمة على Render أضف:

```text
OPENAI_API_KEY=مفتاح OpenAI الجديد
APP_ACCESS_CODE=كود دخول قوي تختاره
OPENAI_MODEL=gpt-5.4-nano
DAILY_REQUEST_LIMIT=50
```

لا تجعل `OPENAI_API_KEY` عاما ولا تكتبه داخل GitHub.

## 4. أوامر Render

Build Command:

```text
npm install
```

Start Command:

```text
npm start
```

## 5. بعد النشر

افتح الرابط الذي يعطيه Render، ثم:

1. أدخل كود الوصول.
2. اضغط حفظ.
3. أرسل سؤالا قصيرا.
4. راقب Usage في OpenAI Platform.

## 6. ملاحظات أمان

هذه نسخة عامة أولية بكود وصول واحد. قبل فتحها لجمهور واسع نحتاج:

- أكواد متعددة أو حسابات مستخدمين.
- حدود استخدام لكل مستخدم.
